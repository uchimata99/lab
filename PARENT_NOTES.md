# המעבדה — עיבוד הערות הורה (הוראות ל‑Claude)

מסמך זה מסביר ל‑Claude מה לעשות כשמקבלים מקבץ **"הערות הורה"** מתוך האפליקציה.
קראי אותו יחד עם `CLAUDE.md` (שם כל העקרונות והאיסורים — **חובה לקרוא לפני כל שינוי**).

## ההקשר
- "המעבדה" היא אפליקציית לימוד אינטראקטיבית בעברית, בקובץ יחיד ועצמאי: `index.html`.
- מתארחת ב‑GitHub Pages במאגר `uchimata99/lab`. לעדכן = להחליף את `index.html`, לעשות commit ו‑push; הדף מתעדכן תוך כדקה.
- כל תוכן הלימוד נמצא במערך `CURRICULUM` שבתוך `index.html` (9 נושאים, 20 שיעורים, בסבב כימיה ← פיזיקה ← מתמטיקה).

## מבנה ההערות שמגיעות מההורה
ההורה משתף טקסט שמתחיל בשורה:

```
הערות הורה — המעבדה
```

ואחריו בלוקים בפורמט:

```
[תחום · נושא · שיעור N (כותרת השיעור) · קטע]
• תיאור מה לתקן
```

ה"קטע" הוא אחד מ‑: `קריאה · דף K` / `התנסות` / `שאלות`.
לעיתים יצורף גם **צילום מסך עם סימוני עט** (בצבע אדום‑ורוד) שמצביעים על האזור המדויק.

## מה לעשות עם כל הערה
1. אתרי בקוד את המיקום: חפשי ב‑`CURRICULUM` את הנושא והשיעור לפי הכותרת, ובתוכו את הדף (`read[]`), ההתנסות (`interactive`) או השאלה (`questions[]`) הרלוונטיים.
2. החילי את התיקון שביקש ההורה, תוך שמירה קפדנית על העקרונות שב‑`CLAUDE.md`:
   - עברית בלבד (למעט סמלים מדעיים ונוסחאות), קובץ יחיד ללא תלות חיצונית ובלי שלב בנייה.
   - חילוק תמיד כפס שבר אופקי; גופנים וערכת הנושא הכהה‑חמה; כיבוד `prefers-reduced-motion`.
   - בטיחות ילדים, תוכן מותאם גיל (כיתות ה׳–ו׳).
3. אם צורף צילום מסך — היעזרי בסימונים לזיהוי האזור.

## אימות לפני כל commit (חובה)
1. **בדיקת תחביר:** לחלץ את תוכן תגית ה‑`<script>` ולהריץ עליו `node --check` (או שקול‑ערך).
2. **בדיקת ריצה:** לדמות מעבר על כל 20 השיעורים ולוודא שאין שגיאות זמן‑ריצה (התקדמות, בנקים, רמות אתגר, הישגים, מסך הורה, תרגול חזרה).

## סיום
- commit עם הודעה תיאורית בעברית, ואז push. GitHub Pages יתעדכן לבד.
- **אל תיגעי** בנתוני ההתקדמות של הילדה ובהערות ההורה (מפתחות `lab_*` ב‑localStorage) — אלה נתוני זמן‑ריצה, לא חלק מהקוד.
